3주차에는 설계(Deployment Diagram)에서 정한 범위를 실제로 구현해 보고, `tk-*.log` 기반으로 **에러 감지 → 로그 묶음 → PMS 티켓 생성(첨부 포함)** 까지의 최소 동작을 확인했다. 구현은 PMS Desktop client(`sgtk`) 환경 전용으로 백그라운드 앱(`tk-incidentreporter`) 형태로 만들었다.

> `sgtk` 는 PMS Desktop 개발을 위한 환경(프레임워크) 이며, [오픈소스](https://github.com/shotgunsoftware)로 공개되어있다.

## 전체 구조 (참고 이미지)

아래 이미지는 배포 다이어그램으로, 클라이언트(엔진/sgtk/DCC)와 중앙 PMS 관계, 그리고 내가 개발할 범위(`tk-incidentreporter`)를 보여준다. 개발환경을 쉽게 보여주고자 시각화 하였다.

![toss-observability-deployment.png](/assets/images/journey/toss-observability-deployment.png)

## 이번 주 목표(MVP 스코프)
* 멀티 OS(Windows/macOS/Linux)에서 공통 방식으로 지정 로그 경로 하위의 `tk-*.log`를 스캔하고 tail/follow 한다.
  * Windows: `%APPDATA%\Shotgun\logs\`
  * Linux: `~/.shotgun/logs/`
  * Mac: `~/Library/Logs/Shotgun/`
* 로그 레벨 중 `CRITICAL`/`ERROR`를 감지하면, **엔진 단위로 로그를 묶고 실행 환경 정보를 수집**한 뒤 PMS에 첨부하여 **티켓을 자동 생성**한다.
* signature 기반 중복 억제(cooldown)를 적용해 동일 이슈의 반복 티켓 생성을 방지한다.
* 최소 동작(MVP)으로 엔드투엔드 흐름(감지 → 수집/묶음 → 첨부 → 티켓 생성)을 검증한다.

## 핵심 컴포넌트

- **SingletonLock** (`singleton.py`)  
  동일 호스트에서 에이전트가 중복 실행되는 것을 방지. QLocalServer 기반으로 단일 인스턴스 보장.

- **TailWorker** (`tail_worker.py`)  
  로그 디렉토리를 glob으로 스캔하고 파일별 오프셋을 유지하며 `readline()`으로 신규 라인을 감지. 롤오버/파일축소 시 재열거·재오픈 처리.

- **Matcher** (`matcher.py`)  
  정규식 기반으로 `ERROR|CRITICAL` 우선 감지, 키워드(예: failed/exception/traceback 등) 보조 감지. 감지 시 정규화된 **signature**(중복 판정용)를 생성.

- **AgentController** (`agent.py`)  
  TailWorker에서 들어온 라인을 받아 Matcher로 판단하고, cooldown 로직으로 중복 억제 후 Uploader 호출.

- **Uploader** (`uploader.py`)  
  sgtk가 노출하는 API 핸들(`app.shotgun` 또는 `engine.shotgun`)을 사용해 Ticket을 생성하고 로그 파일을 첨부. 동기 업로드·재시도(백오프) 지원. 티켓 description에 매칭 라인·엔진 정보·signature·환경정보 포함.

- **Bootstrap / Autostart** (`bootstrap.py`, `autostart.py`)  
  앱 초기화와 AgentRunner 시작/중지 로직.


## 설계상의 핵심 결정들

- **파일 기반 감지(tail)**: 프로세스 후킹 대신 파일시스템(tail)으로 통일해 OS·DCC에 관계없이 동일한 방식으로 동작하도록 설계했다.  
- **엔진 단위 로그 묶음**: `CRITICAL/ERROR` 레벨 발생시 에러 전후의 맥락을 주기 위해 엔진 단위 로그 묶음을 전송하도록 했다. 
- **signature + cooldown**: 동일 이슈의 반복으로 티켓이 폭주하는 걸 방지하기 위한 간단한 dedupe 메커니즘을 도입했다.  
- **로그 롤오버**: 로그 파일이 rotate 되면 tail 핸들이 무효화된다. 이를 대비해 폴더 재열거와 inode/크기 비교 후 재오픈 로직을 넣었다.
